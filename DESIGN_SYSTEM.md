# NIS Math (kodi-web) — Design System

> Зафиксировано: 2026-06-22. Документирует то, что РЕАЛЬНО существует в коде сегодня (`lib/app/theme.dart`, `lib/app/colors.dart`, `lib/shared/widgets/*`), а не желаемое. Глобальное правило: **перед любой UI-работой читать этот файл**. Токены — закон (пол): новые экраны берут цвета/радиусы/отступы отсюда, не плодят свои литералы. Композиция и характер экрана — свобода дизайнера (потолок).
>
> Приложение: Flutter Web, образовательная платформа по математике для учеников НИШ. Аудитория — школьники. Тон — дружелюбный, поддерживающий, светлый. Это НЕ плотный аналитический дашборд: воздух, крупные тапабельные элементы, мягкие карточки, эмодзи как иконки разделов — уместны.

## Дух

Светлый, тёплый, ободряющий. Один доминирующий цвет (синий), палитра-светофор для прогресса (зелёный/оранжевый/красный), мягкие тени вместо жёстких границ, скруглённые углы 14–20px. Прогресс ученика — главный визуальный сигнал на каждом экране (ring chart, mastery-бары, трёхсегментные полосы). Праздновать успех (✓, mastered-бейдж, 🔥 streak, combo-градиент), мягко подсвечивать ошибку — без наказующего тона.

Все цвета централизованы в `AppColors` (`lib/app/colors.dart`) — «заменяет 25+ разбросанных `Color(0xFF...)`». Новый цвет — сначала проверь, нет ли его там; новый оттенок — добавляй в `AppColors`, не хардкодь в виджете.

### Адаптив (`lib/shared/utils/responsive.dart`)

Базовая ширина дизайна — **414px** (iPhone Pro Max). Приложение mobile-first, центрируется на широких экранах через `ConstrainedBox(maxWidth: 500)`.

- `rs(context, size)` — пропорционально масштабирует размер шрифта/иконки вниз на узких экранах (≤414px = 1:1).
- `rp(context, padding)` — ступенчато режет отступы: `<360px → ×0.65`, `<390px → ×0.8`.
- `isSmallScreen(context)` — `width < 375`.

**Правило:** размеры шрифтов и отступы в карточках задавать через `rs()`/`rp()`, а не голыми числами, иначе на узких телефонах вёрстка ломается.

## Типографика (фактическая шкала)

Кастомного `TextTheme` нет — используются inline `TextStyle`. Наблюдаемая шкала (база 414px):

| Уровень | Размер | Вес | Где |
|---|---|---|---|
| Hero-заголовок | 26 | bold | onboarding |
| Заголовок прогресса | 22 | bold | hero, % в section |
| Текст задачи | 17 | w400, height 1.6 | `ProblemCard` / `MathText` |
| Заголовок результата | 17 | bold | `ResultCard` |
| Название раздела | 16 | w600 | section, input |
| Body / решение | 14 | w400–600, height 1.5 | solution, topic |
| Подпись | 11–13 | w400–600 | counters, hints, муты |

Веса в ходу: `w400 / w500 / w600 / w700 / bold`. Контраст весов умеренный (нет экстремальных скачков — это ОК для дружелюбного учебного UI, не для «инженерного» дашборда).

## Рендеринг математики (LaTeX)

Через `flutter_math_fork: ^0.7.4`, единый виджет `MathText` (`lib/shared/widgets/math_text.dart`). Используется во всех местах с контентом задач: `ProblemCard`, `ResultCard` (ответ + решение).

- Текст с инлайн-формулами: сегменты `$...$` рендерятся через `Math.tex(... mathStyle: MathStyle.text)`, остальное — обычный `TextSpan`. Смешанный текст+формула в одном `Text.rich`.
- **Авто-конвертация plain-текста в LaTeX** (`_convertToLatex`): дроби `1/3 → \frac{1}{3}`, смешанные числа `2 3/4`, степени `x^2`, корни `√25 → \sqrt{25}`, `⋅ → \cdot`. То есть авторам контента НЕ обязательно писать LaTeX вручную для базовой математики.
- Формулы рендерятся на **1.15× крупнее** окружающего текста (выравнивание по середине строки) — чтобы дроби/индексы читались.

**Правило:** любой пользовательский текст, который может содержать математику, выводить ТОЛЬКО через `MathText`, не через `Text`.

## Ключевые компоненты (источник правды по стилю)

- **`ProblemCard`** — белая карточка (radius 16, мягкая тень), сверху строка мета: плашка темы (blue-50 фон, primary текст), «точки сложности» (4 точки, заполнены primary), счётчик-pill. Контент через `MathText` 17px.
- **`AnswerInput`** — белая карточка (radius 14), `TextField` с заливкой surface и focus-border `accentColor width:2`; `FilledButton` (Отправить) + `OutlinedButton` (Пропустить) + icon-кнопка (Пожаловаться). `accentColor` параметризуем (практика=синий, можно менять под контекст).
- **`ResultCard`** — подложка успех/ошибка (emerald-50 / red-50) с цветной рамкой 1.5px; иконка `check_circle` / `cancel`, mastered-бейдж, блок верного ответа, блок решения (полупрозрачный белый), **mastery-бар** (LinearProgressIndicator, цвет по порогам ≥0.7 success / ≥0.4 warning / иначе primary).
- **`HeroCard`** (дашборд) — синий градиент, белый текст, `RingChart` (CustomPaint, кольцо прогресса) + 🔥 streak. Главный «signature»-элемент дашборда.
- **`SectionCard`** — раскрывающаяся карточка раздела: эмодзи-иконка, %-цвет по порогам (≥75 зелёный / ≥60 оранжевый / красный), трёхсегментный бар (green/red/gray), `AnimatedCrossFade` для списка тем.
- **`OnboardingView` / `StepCard`** — пустое состояние нового ученика: градиентная иконка-аватар, 3 пронумерованные шаг-карточки с цветными иконками, CTA «Начать диагностику».

## Состояния компонентов

| Состояние | Как реализовано сегодня |
|---|---|
| **Loading** | `CircularProgressIndicator` по центру (auth-гейт в `main.dart`, страницы фич). HTML-сплэш в `web/index.html` до загрузки Flutter. |
| **Error** | `ErrorView` — иконка `error_outline` (48, error-цвет) + сообщение + `FilledButton` «Повторить» (`onRetry`). Тексты ошибок локализованы (`error_l10n.dart`). |
| **Empty** | `OnboardingView` для нового ученика (нет mastery); прочерк `—` для непротестированных тем. |
| **Success** | `ResultCard` (зелёный), mastered-бейдж, combo-градиент анимация, streak. |
| **Hover / focus** | `InkWell` ripple на тапабельных карточках; `TextField` focusedBorder = `accentColor` 2px. Web-специфичный hover отдельно не стилизован (полагаемся на Material defaults). |
| **Анимации** | `AnimatedRotation` (стрелка раскрытия), `AnimatedCrossFade` (раскрытие тем 250ms), fade-in результата 400ms, combo 600ms, `BouncingScrollPhysics`. |

**Правило:** каждый новый интерактивный компонент обязан иметь loading / error / empty / success-состояния — переиспользуй `ErrorView` и существующие паттерны, не изобретай свои.

## Локализация

Двуязычный (`ru`, `kk`) — `AppLocalizations` (`lib/l10n/`). Тексты в UI НЕ хардкодить: всё через `l.<key>`. Поле `NisApiClient.lang` (`kk → 'kz'`) переключается через `LocaleBloc`.

## Текущее состояние и рекомендации

Честная оценка для бэклога будущей полировки. Токены остаются законом; ниже — что выглядит дефолтно и стоит усилить ПОВЕРХ них.

**Что хорошо:**
- Палитра централизована в `AppColors` — нет разбросанных литералов, рефактор цвета = одна правка.
- Тёплый scaffold-фон `#FAF9F6` (не дефолтный чистый белый) — приятная мелочь, уже анти-слоп.
- Прогресс как сквозной visual signal (ring, бары, светофор) — у приложения есть характер и понятная иерархия.
- Mobile-first адаптив через `rs()`/`rp()` — продуман для узких телефонов.
- LaTeX-пайплайн с авто-конвертацией — сильная предметная фича.

**Что читается как дефолт / generic AI-slop (бэклог):**
- ⚠️ **Шрифт — Roboto** (дефолт Flutter Web; кастомный шрифт нигде не подключён: ни `google_fonts`, ни `fontFamily`, ни `assets/fonts`, ни `<link>` в `index.html`). Это прямо нарушает правило владельца «NEVER Inter/Arial/Roboto». **Рекомендация:** подключить выразительный шрифт — для дружелюбного учебного приложения подойдут округлые гуманистические гротески (например **Nunito**, **Lexend** — последний оптимизирован под читаемость у школьников, **Onest**, **Manrope**) для UI-текста + чёткий шрифт под цифры/проценты. Завести `TextTheme` в `theme.dart`, чтобы шкала была токеном, а не россыпью inline-`TextStyle`. Это самый высокоэффективный шаг полировки.
- Базовый лук «синий primary + белые карточки + мягкая тень» — один из дефолтных AI-образов. Сами токены легитимны (учебное приложение, светлый тон), но выразительность добирается ПОВЕРХ: фирменный шрифт, один запоминающийся signature-элемент на экран (hero-кольцо это уже умеет — развить идею на практику/экзамен), чуть смелее типографский контраст в заголовках.
- Тени заданы вручную в каждом виджете (`0.04–0.05 alpha`), при этом `cardTheme` в теме даёт elevation 0 + border — две разные модели карточки сосуществуют. **Рекомендация:** свести к одному решению (вынести «card shadow» в общий `BoxDecoration`-хелпер или довериться `cardTheme`), чтобы карточки были консистентны.
- Радиусы слегка вразнобой (8/12/14/16/20). Не критично, но стоит зафиксировать 2–3 ступени как токены и придерживаться.

**Граница ответственности:** токены выше (цвета, радиусы, отступы, шрифт-шкала, состояния) — **пол, который нельзя опускать** (берём как есть, не плодим параллельные литералы). Композиция экрана, выбор signature-элемента, ритм и характер — **потолок, свобода дизайнера**. «Modern, clean» — нулевая информация; направление задаём предметно: «дружелюбный учебный трекер прогресса для школьника, где победа над темой — это маленький праздник».
