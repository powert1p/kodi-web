## Оверрайд для webapp/ (React PWA тренажёра)

- **Бриф** = `DESIGN-V6.md` текущего run'а (+ webapp/DESIGN_SYSTEM.md как действующий канон до его обновления). Ось `brief_fit` читать как «соответствие регистру/брифу v6 + задаче ЭКРАНА»: это приложение для ребёнка 10-13, а не лендинг — «дорого-взросло-премиально» при потере тепла/ясности = провал оси, и наоборот «kiddie-детсад» = провал.
- **Скриншоты** = мобильные, **390×844 full-page** (главный вьюпорт продукта); 1280 не требуется в R1-концептах, в React-раундах — 768+ как «планшет в центре» (§4 канона). Судья обязан смотреть ВСЕ экраны варианта, не только hub.
- **«Визуальный панч»** для приложения = первый вьюпорт КАЖДОГО экрана впечатляет и хочется остаться; глубина/иллюстрация/signature читаются сразу. «Аккуратные карточки на белом» = 5-7 (провал).
- **Craft-оси авто-потолки:** внутренние идентификаторы (snake_case) в UI · красный для ошибок · учебный текст <18px · тач <44px · второй primary-CTA на экране · фейковые данные в декоре (нечестные шкалы/числа) → ось craft ≤4 автоматом.
- **Signature** должен быть ОДИН, сквозной через экраны (hub/drill/srez), работать на мобиле в полноценной форме и на ЧЕСТНЫХ данных (реальное число ступеней/точек).
- **Рендер:** R1 (статичные HTML-концепты) — `file://` через render-скрипт run'а; R2+ (React) — vite localhost:5173 или прод-подобный билд, console чистая, без горизонтального скролла.
- **Детектор** = `node ~/.claude/skills/impeccable/scripts/detect.mjs <html|url>` — 0 критичных флагов (входной билет в судейство).
- Дельты и починки флоу → `docs/loops/FLOW-CHANGELOG.md` (kodi-web), кросс-переносимые уроки дизайн-флоу → `~/Documents/design-lab/FLOW-CHANGELOG.md`.
