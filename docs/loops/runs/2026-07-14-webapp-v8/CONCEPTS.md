# Round 0 — три clean-slate направления

## A. «Пульс решения»

**Spatial model:** насыщенная cobalt-сцена; рабочая задача живёт на светлой «линзе», которая
режет экран диагональю, а не складывается в card stack. Desktop — широкая reasoning-stage;
mobile — тот же разрез во всю ширину.

**Signature:** `proof trace` — световая линия реальных шагов. Она стартует из формулировки,
делает излом в месте прошлой ошибки и приводит к текущему input. Hub показывает отдельные
trace-фрагменты очереди, drill — полный ход, srez — один незавершённый импульс.

**Type/atmosphere:** узкий технический display с кириллицей + спокойный humanist body;
cobalt / bone / volt-yellow, tangerine зарезервирован за Кёди. Полупрозрачные оптические
слои и custom macro asset дают глубину без «карточек с тенью».

**Риск:** может стать слишком игровым/киберпанковым; запрещены neon glow spam, HUD-labels
и near-black QARA-копирование.

## B. «Поле доказательства»

**Spatial model:** светлая typographic canvas с огромными математическими полями и боковыми
заметками; контент организован как доказательство, а не как dashboard. На desktop задача и
рабочий шаг сталкиваются по асимметричной baseline-сетке; mobile сохраняет широкую margin-ось.

**Signature:** `proof marks` — система скобок, подчёркиваний и координатных выносок, которая
связывает условие, ошибочный фрагмент и следующий шаг. Данные — реальные номера шагов.

**Type/atmosphere:** expressive serif/condensed display + нейтральный sans; bone / deep ink /
electric blue / signal coral. Raster asset — macro-типографическая текстура без текста.

**Риск:** может стать взрослым magazine-layout и замедлить решение; запрещены newspaper
all-caps, teacher-red и декоративные annotation без функции.

## C. «Координатная студия»

**Spatial model:** полноэкранные цветовые поля и крупные геометрические apertures; каждый
экран — одна сцена, внутри которой secondary content открывается слоями. Desktop использует
горизонтальную дугу, mobile — вертикальную, без превращения в bottom-card list.

**Signature:** `focus orbit` — орбита вокруг текущей точки решения. На hub она ранжирует
ошибки по близости к фокусу, в drill обнимает активный шаг, в srez показывает единственный
checkpoint. Количество меток всегда совпадает с данными.

**Type/atmosphere:** geometric display + readable grotesk; ultramarine / ice / warm coral /
black. Code-native vector field вместо фотографии.

**Риск:** блоки могут уйти в kiddie/edtech; запрещены primary-color rainbow, clay, emojis и
равномерная Bento-сетка.

## Финальный выбор

**A «Пульс решения»** выбран всеми тремя blind judges. Прототип получил `NOT_YET`, поэтому
в React переносится направление, а не общий HTML-шаблон. Обязательные исправления signature,
motion, contrast и register зафиксированы в `round-0/RESULT.md` и `DESIGN-V8.md`.
