# DESIGN v6 — «Тетрадь чемпиона» (бриф редизайна мобилки, читают maker'ы + судьи)

**Surface:** мобильное PWA-приложение (390×844 first), НЕ лендинг. Экраны концепта: hub, drill, srez.
**Product:** Kodi — AI-тренажёр математики оффлайн-центра. Ребёнок 10-13 лет готовится в НИШ: фотографирует решение из тетради, разбирает ошибки по ступеням-лесенке, спрашивает Кёди (оранжевый маскот-мандаринка). Тетрадь в клетку — родная среда продукта: математика реально решается ручкой в тетради рядом с телефоном.
**Audience:** ребёнок 10-13 (НЕ детсад — с 12 лет kiddie-эстетика вызывает отторжение), казахстанская школа, амбиция «поступлю в НИШ». Родитель видит экран мельком — должен считать «серьёзно и по-доброму».

## Register lock (перечитывать перед каждым self-check)
**Регистр: ТЁПЛАЯ МАСТЕРСКАЯ ЧЕМПИОНА** — warm-craft бумаги и маркера × спокойная спорт-энергия пути к цели. «Уголок подготовки, где тебя ждут»: тепло, вещественно, сфокусированно.
Этот регистр ЗАПРЕЩАЕТ дефолты дизайн-флоу:
1. **Тёмный премиум** (near-black, люкс-минимализм, editorial-шик Arc'teryx) — это приложение живёт СВЕТЛЫМ и тёплым.
2. **Взрослая киношная драма** — hero-фото «величественных вершин на закате» не наш масштаб; наша драма — прочерченный маркером путь в клетчатой тетради.
3. **Kiddie/цирк** — радуга, пузырьковые шрифты, эмодзи-дождь, «мимими» — отторжение у 12-леток и родителей.
4. Неон/брутал/SaaS-градиент (purple-blue) — всегда запрещены.

## Что v6 СНИМАЕТ из v5 (осознанные решения дизайн-директора; остальной v5-канон в силе)
1. **«Только Golos Text»** → снято. Добавляется ХАРАКТЕРНЫЙ display (см. Type). Golos Text остаётся body.
2. **«NEVER градиентные фоны» / «плоскость + 1px stroke только»** → смягчено: разрешены **тональные тёплые paper-washes**, **тетрадная клетка** как живая текстура, **слои бумаги со светом** (мягкий крафт-lift у активной карточки — один стиль тени, не спам). SaaS-градиент фиолетово-синий — по-прежнему запрещён.
3. **«Иллюстраций нет»** → снято: генерённые иллюстрации обязательны (см. Assets).
Не тронуто из v5: `brand`-оранж = ТОЛЬКО действие · никогда красный (ошибка = тёплый амбер) · никогда тёмная тема · один primary-CTA на экран · учебный текст ≥18px · тач ≥48px · Кёди-протокол (hi/thinking/oops/celebrate, один на экран) · никаких внутренних id в UI · max-width 480 по центру на ≥768.

## Colors (доминанта + один акцент)
- **Доминанта:** тёплая бумага — семейство от `#FAF7F2` (свет) до `#F3EDE2`/`#EFE7D8` (глубокие слои); чернила `ink #201D17` (тёплый почти-чёрный, НЕ #000), text `#3D3A34`, muted `#8A857C`.
- **Акцент ОДИН:** kodi-оранж `#FF8C00` (hover `#E67E00`, подложка `#FFF3E2`) — действие, прогресс, маркер маршрута. Не действие → не оранжевый.
- Служебные: success `#2E9E5B`, амбер «не сошлось» `#B7791F` (+soft-пары). Красного НЕТ нигде.
- Глубина — слоями тёплой бумаги и светом, не серыми тенями. Клетка тетради — тонкой линией `#E4DCCB`-семейства, чуть теплее stroke.

## Type (пара двух семейств; кириллица обязательна, каз. Ң — проверить)
- **Display** — геометричный/уверенный характерный шрифт с РОДНОЙ кириллицей: числа-герои («12», «3/5», «1200 ₸»-подобные), заголовки экранов, ступени-номера. Кандидаты на верификацию (порядок = моя приоритезация): **Unbounded** (вариативный 200-900, спорт-характер), **Golos UI VF в 900 сверх-крупно** (запасной, «свой»), **Days One**, **Russo One**. Анти-пул судей (Playfair/Cormorant/Yeseva/Lora/Oswald и Inter/Roboto) не брать. Верификация ДО выбора — метод FLOW.md §2.5: dual-fallback spread по слову «ВЕРШИНА» + отдельно глиф **Ң** (U+04A2); молчаливый fallback = провал.
- **Body** — **Golos Text** (уже self-hosted в проекте, оставить): учебный текст ≥18px, реплики Кёди, подписи.
- Контраст уровней ≥3x (display-числа 40-64px против body 16-18px), веса 400↔800-900. Числа: tabular, не рвать по строкам.

## Signature (ОДИН, сквозной — обязателен)
**«Маршрут маркером по клетке»**: живая оранжевая линия — как будто ребёнок прочертил маркером путь по клетчатой странице — ведёт через КАЖДЫЙ экран и несёт ЧЕСТНЫЙ прогресс:
- **hub** — маршрут через список ошибок: пройденные участки прочерчены, текущая точка пульсирует, впереди — пунктир; 12 ошибок = 12 честных отметок.
- **drill** — лесенка ступеней КАК вертикальный участок маршрута: 4 ступени = 4 отметки высоты, верная ступень «защёлкивается» и линия дорисовывается вверх (stroke-dashoffset).
- **srez** — 12 точек пути, «ты на 1-й»; финал — флажок.
Требования: командует в первом вьюпорте каждого экрана (не задворки), работает на мобиле в полноценной форме, данные честные (число отметок = числу реальных шагов/задач), рисуется motion'ом при входе (reduced-motion → мгновенно). Трактовка (толщина, характер линии, отметки-крестики/флажки/точки) — свобода maker'а.

## Экраны концепта (реальный контент — использовать ДОСЛОВНО, это прод-данные)
**hub.html** — «Срез на сегодня»: hero «12 ошибок ждут» + Кёди hi + прогресс «Разобрано сегодня 0/12 · 0%» + CTA «Разобрать первую»; список ошибок (сложные сверху), карточки: «Текстовые задачи на систему уравнений» / «2 тетради и 3 ручки стоят 240 тенге. 1 тетрадь и 2 ручки — 140 тенге. Сколько стоит ручка?» / «в прошлый раз: 1»; «Концентрация (задачи на смеси)» / «Из 300 г 20%-го раствора выпарили 100 г воды. Какова концентрация?»; «Площадь комбинированных (составных) фигур» / «Из прямоугольника 10×8 вырезали квадрат 3×3. Найдите площадь оставшейся фигуры.»; «Модульные уравнения» / «Уравнение |x − 3| = 5. Найдите произведение корней.»; нижняя таб-навигация «Срез · Прогресс».
**drill.html** — «Работа над ошибкой · Текстовые задачи на систему уравнений · 1/4» + прогресс; Кёди thinking «Уровень 1 · с нуля — Разберём тему вместе, по одному маленькому шагу. Спешить некуда.»; карточка ЗАДАЧА (текст выше) + «✏️ В прошлый раз получилось 1 — разберёмся, где сбилось»; раскрывашка «Как решать?» ОТКРЫТА в концепте: «**Метод** — Введи для каждой неизвестной величины свою букву, запиши два уравнения из условия, и реши систему сложением или подстановкой — по одной подсказке ответ не угадать. **Пример** — Сумма 30, одно в 2 раза больше другого: x+2x=30 → 3x=30 → x=10. **Ловушка** — составляют только одно уравнение из двух условий и не могут найти оба числа однозначно.»; переключатель «Ввод | По тетради»; лесенка: шаг 1 АКТИВЕН «Составление системы уравнений по условию задачи — Пусть t — цена тетради, r — цена ручки. Составь систему: 2t + 3r = 240 и t + 2r = 140.» + поле «Твой ответ» + «Проверить»; шаги 2-4 заперты: «Решение линейного уравнения», «Решение системы методом подстановки», «Решение линейного уравнения»; ниже — «Спросить Кёди» с фрагментом чата: ученик «не понимаю с чего начать, тут же две неизвестные» → Кёди «Отличное замечание! Здесь действительно две вещи, чьи цены мы не знаем. Когда у нас две неизвестные, удобно дать им имена — буквы. Как бы ты назвал цену тетради и цену ручки?»
**srez.html** — «Мини-срез · 1 из 12» + прогресс; «Тема: Цифры числа (разряды, количество)»; задача «Найдите двузначное число, если оно равно удвоенному произведению своих цифр.»; поле «Твой ответ» + «Проверить»; таб-бар.

## Углы трёх maker'ов (каждому — свой, регистр один на всех)
- **K-A «Тетрадь»:** клетка и маркер ведут всё — рукописные подчёркивания, крафт-слои бумаги, стикер-отметки; иллюстрации-натюрморты письменного стола.
- **K-B «Экспедиция»:** вертикаль высоты и цель НИШ — альтиметрия ступеней, отметки высоты, номер-гиганты display'ем, спорт-ритм; тепло сохраняется бумагой.
- **K-C «Мастерская Кёди»:** Кёди — хозяин пространства: сцены-иллюстрации, «полки» слоёв бумаги, живые реакции маскота, максимум характера при взрослой строгости сетки.
