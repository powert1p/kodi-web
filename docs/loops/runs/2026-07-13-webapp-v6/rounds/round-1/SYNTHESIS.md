## Решение: гибрид «K-B скелет + прививки»
Базу R2 берёт **K-B** (сильнейшие типографика/панч/композиция: Unbounded-гиганты, альтиметр-структура, лучший detect/craft-каркас), с обязательными прививками:
- **из K-C**: живая РУКОПИСНАЯ кривая маршрута (stroke-dashoffset «перо рисует», не ruler-straight таймлайн) + голос Кёди-хозяина (реплики от Кёди, «Ты здесь»-человечность);
- **из K-A**: craft-дисциплина — формат-нейтральные плейсхолдеры, цветокод МЕТОД/ПРИМЕР/ЛОВУШКА + формулы-чипы (≥18px), ink-пары контраста (`--brand-ink #9A5B00` на светлом — уже в K-B).

## Типографика (зафиксировано, спор судей разрешён probe'ом)
Display **Unbounded** (SELF-HOST woff2-сабсет с кириллицей — гуглохостинг кириллицу НЕ отдаёт; сабсет K-B с рендер-доказательством spread 0px) + body **Golos Text**. Ң у Unbounded НЕТ → каскад `'Unbounded','Golos Text',sans-serif`. В R2-гейте оркестратор САМ гоняет dual-fallback spread на живом приложении (self-report не принимается).

## Консенсус-директивы R2 (каждая = дельта ≥2 судей; нарушение = провал раунда)
1. **Hub, первый вьюпорт:** число-герой и маршрут ЖИВУТ ВМЕСТЕ — «12» стоит НА маршруте (первый изгиб кривой проходит через hero), CTA «Разобрать первую» виден без скролла. Не «number-app сверху, путь ниже».
2. **Маршрут везде** — рукописная кривая маркером по клетке, рисуется stroke-dashoffset (и на srez тоже — у K-B/K-C там было слабее/несогласовано), честные отметки (12/4/12), «+8» хвост — явные честные точки, не декор-пыль.
3. **Drill, середина:** активная ступень — САМАЯ ТЯЖЁЛАЯ масса экрана (масштаб + свет/крафт-lift); запертые ступени — «призраки» (компакт, низкий контраст); теория «Как решать?» и чат Кёди СВЁРНУТЫ по умолчанию и в раскрытом виде тонированы тише активной ступени. Первому вьюпорту drill — сильный якорь (дробь-альтиметр «1/4» + активная ступень).
4. **Ноль утечек в UI-хроме:** плейсхолдер никогда не содержит чисел/ответов («Запиши систему», «Твой ответ»); inputmode соответствует типу ответа ступени (система/выражение → text, число → numeric). (Данные ступеней местами сами содержат ответ в instruction — фикс банка отдельной волной, UI утечек НЕ добавляет.)
5. **Учебный текст ≥18px строго** (метод/пример/ловушка, реплики Кёди, инструкции ступеней, формулы-чипы); тач ≥48px все контролы; обращение везде «ты».
6. **Чат:** пузырь ученика — тёплая brand-soft подложка, НЕ ink/near-black.
7. **Кёди-протокол на каждом экране** (hub hi · drill thinking · srez — присутствие в интро; oops при 2+ фейлах; celebrate в closure), ошибку формулирует Кёди.
8. **Оранж-дисциплина:** оранж = маршрут/прогресс/primary-CTA; пилюли, иконки таб-бара, бейджи — ink/нейтраль/soft.
9. **Таб-бар — часть системы:** активный таб = отметка маршрута (не генерик-иконки).
10. **Иллюстрация hub — осознанная:** тёплая явная полоса с честным scrim под текстом (или не ставить вовсе); текст поверх фото — только со сплошным scrim (AA).
11. **Motion:** stagger-reveal списков при входе + draw маршрута; prefers-reduced-motion → мгновенно; bounce запрещён.
