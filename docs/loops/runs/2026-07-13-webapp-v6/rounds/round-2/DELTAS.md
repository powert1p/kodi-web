# R2 → R3: консенсус-дельты панели (J1 7.25 · J2 6.94 · J3 7.0 — NOT_YET)

Каждый пункт = паттерн ≥2 судей (или несущий брак одного с проверяемым фактом). R3 = фикс-раунд, один maker. Порядок = приоритет.

## 1. SIGNATURE: маршрут должен быть ВИДЕН и сквозной (J1+J2+J3 — главный провал, директивы №1/№2)
- Линия видима С НУЛЕВОЙ СЕКУНДЫ статично: карандашный след-подложка (тёплый серый, вся траектория) + плотный оранжевый пунктир пройденного (`stroke-dasharray` ~6 8, opacity ≥0.65) + draw-анимация ПОВЕРХ как усиление, не как условие видимости. Скриншот в любой момент должен показывать маршрут.
- Hub hero: первый изгиб кривой заходит ВНУТРЬ hero-карточки — стартовая точка под числом-героем, мимо CTA, к стопу 1 (сейчас рельс сбоку, «4» стоит рядом, не НА маршруте).
- Closure: финальный участок маршрута + флажок вершины у celebrate («Эту ошибку ты победил» = конец пути). Analytics: присутствие маршрута (мини-форма: линия прогресса тем как участки пути), не «точка под таб-баром».
- Графический контраст линии/заливок прогресса ≥3:1 на бумаге (сейчас оранж ~2.0-2.2:1) — затемнить штрих линии (сторона --brand-ink/деп-оранж), CTA-заливки не трогать.

## 2. ВАЛЮТА: ₸ везде (J1+J2+J3)
Моки говорят «1200 ₽»/«Сколько рублей…» (hub/mock.ts, drill/mock.ts шаги) при «800 ₸» в closure. Продукт — Казахстан/НИШ: везде тенге («₸», «тенге»), суммы согласованы между экранами. Моки формой = прод-данные (в боевом банке — тенге).

## 3. КОНТРАСТ AA (J3 — несущий брак, блокирует СУПЕР; J1 частично)
- `--muted #8a857c` = 3.43:1 на paper → затемнить токен до ≥4.5:1 на paper И surface (все плейсхолдеры «Твой ответ», вторичные подписи, login-сабтайтл 18px).
- Disabled primary «Проверить» 2.98:1 (заметная крупная кнопка srez/drill/closure) → поднять контраст disabled-состояния (ink-пара на приглушённой заливке).
- Мелкий muted-на-цветном перепроверить по экранам.
