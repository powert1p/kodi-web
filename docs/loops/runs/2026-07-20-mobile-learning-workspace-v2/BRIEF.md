# Mobile learning workspace v2 — brief

## Outcome

Ребёнок на телефоне понимает текущую задачу и следующее действие без догадок. Условие,
ответ или фото, AI-проверка, guided step и разговор с помощником живут в одном устойчивом
learning workspace. Ни один успешный typed answer не требует фото.

## Problem evidence

1. Guided API отдаёт только `instruction`; UI подписывает поле «Ответ шага N», но не сообщает,
   что именно и в каком формате требуется написать.
2. Guided submit проверяет `expected_value` через `check_step_evidence`, хотя owner решил, что
   semantic verdict должен принадлежать AI.
3. Tutor-модель не видит условие и active step. Она выбирает один из пяти `move`, а сервер
   заменяет ответ одной из десяти заготовок; поэтому реальный вопрос ребёнка игнорируется.
4. Mobile — уменьшенная версия desktop-сетки: task, context и response разорваны на крупные
   зоны, а sheet помощника забирает экран вместо продолжения текущего действия.

## Three distinct directions

### A. «Рабочая тетрадь» — preferred

- Пространство: одна непрерывная вертикальная лента; условие сверху, активная работа сразу под ним.
- Действие: компактный inline composer объединяет формат ответа, ввод, primary CTA и feedback.
- Помощь: контекстная нижняя панель 68–76dvh; условие/шаг остаются видимым контекстом.
- Desktop: две колонки — условие и рабочая лента; никогда не три конкурирующих панели.
- Арт: тёплая бумага, глубокий forest, один orange action, тонкие notebook-линии.
- Характер: спокойный, собранный, дружелюбный; не школьный журнал и не SaaS dashboard.
- Риск: длинное условие может отодвинуть действие; решается compact task summary после scroll.

### B. «Диалог у доски»

- Пространство: условие закреплено коротким верхним блоком, вся работа идёт timeline-сообщениями.
- Действие: ответ, AI-verdict, guided prompt и tutor reply — последовательные turns.
- Помощь: помощник является участником timeline, отдельного sheet нет.
- Desktop: доска слева, журнал диалога справа.
- Арт: контрастная тёмная доска + светлая лента ответов, orange chalk-mark для действия.
- Характер: живой урок с преподавателем, сильнее ощущается AI-сопровождение.
- Риск: математика превращается в чат, а история быстро становится шумной.

### C. «Маршрут решения»

- Пространство: компактная вертикальная нить из шагов, раскрыт только текущий узел.
- Действие: одно поле внутри активного узла; подтверждённые узлы сжимаются в короткую запись.
- Помощь: ответ помощника появляется как боковая заметка текущего узла.
- Desktop: условие слева, маршрут справа.
- Арт: мягкие sage-сегменты, orange-marker текущего узла, минимум контейнеров.
- Характер: больше progress/reward, ближе к игровому обучению без mascot.
- Риск: путь начинает конкурировать с условием и создаёт ложный progress до mastery.

## Chosen direction

Выбирается A «Рабочая тетрадь». Она лучше всего закрывает owner complaints: не маскирует
непонятный контракт игровой декорацией, сохраняет математику главным объектом и объединяет все
действия в одном месте. От B берём естественный разговорный ответ помощника; от C — только
тонкий индикатор места в guided sequence, без большой карты прогресса.

## Frozen product rules

- Mobile-first 375×844; desktop — осмысленное расширение до двух зон.
- Один текущий вопрос и одно primary action в first viewport.
- Guided step содержит direct prompt, non-answer-bearing format hint и пример на посторонних числах.
- Guided typed evidence получает grounded AI verdict; backend проверяет schema, echo, confidence,
  idempotency и state transition, но не вычисляет semantic verdict скриптом.
- `unsure` и provider failure сохраняют step/draft и не называются математической ошибкой.
- Tutor видит statement, canonical steps, active step и scoped conversation; отвечает на вопрос,
  проверяет мысль или задаёт один следующий вопрос по ситуации.
- Tutor не раскрывает готовый ответ по умолчанию и не принимает решения за ученика.
- Correct final typed answer завершает task без фото.
- Touch targets минимум 44 px; visible focus; page horizontal overflow не более 1 px.
- Предыдущая «Фокус-станция», трёхзонная сетка, giant headings/cards и canned tutor — anti-reference.

## Acceptance scenarios

1. Ребёнок открывает guided step и без внешнего объяснения может ответить на вопросы:
   «что мне сейчас сделать?» и «что написать в поле?».
2. Эквивалентная математическая запись принимается AI; неверный ход получает конкретную,
   не раскрывающую ответ обратную связь и остаётся на том же шаге.
3. В tutor ребёнок спрашивает «что значит из оставшихся?» и получает grounded explanation,
   а не «назови следующий шаг».
4. На 375×844 condition, current action, input и CTA не перекрываются; keyboard не ломает flow.
5. На 1280×900 математика доминирует; нет трёх равноценных колонок.
6. Loading, offline, provider error, incorrect, unsure и correct имеют ясное локальное продолжение.
