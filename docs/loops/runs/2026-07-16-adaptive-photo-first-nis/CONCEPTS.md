# Concept directions — одинаковая truth, разные spatial/art systems

Общий сценарий fixture для честного сравнения: Саша, 6 класс; диагностика 7/12; уверенно решает
дроби, теряет модель в задачах на отношение; первым приоритетом становится «Отношения и
пропорции». Во всех направлениях показываются одни и те же route, самостоятельная задача и
photo-first actions.

## A — Атлас маршрута

1. **Register:** уверенный; запрещены dashboard soup, candy-map и mascot-first composition.
2. **Direction/spatial:** один непрерывный вертикальный атлас от формата NIS к текущему узлу;
   mobile идёт по оси, desktop раскрывает route-map слева и активный объект справа.
3. **Typography:** M PLUS Rounded 1c для коротких display/чисел, Onest для body и математики.
4. **Palette:** oatmeal `#F4EBDD`, forest `#173F35`, orange `#F08A28`, sage `#DCE7D8`, oxide
   `#A64F39`; orange только active/action.
5. **Signature:** `Нить доказательств` связывает официальный блок → проверенный навык → текущую
   задачу → сохранённое доказательство; это не generic progress bar.
6. **Assets:** один code-native compass/seed mark в onboarding; белка только в спокойной note
   после результата, не на task screen.
7. **Layout/rhythm:** открытая бумажная плоскость, крупные интервалы 24/40/64, одна активная
   станция; desktop не растягивает mobile cards.

## B — Рабочая тетрадь

1. **Register:** уверенный; запрещены официальный школьный портал, табличный dashboard и
   декоративная рукопись.
2. **Direction/spatial:** приложение как раскрытая умная тетрадь; на mobile один лист/разворот,
   на desktop два функциональных поля: контекст на полях, действие на чистом листе.
3. **Typography:** Literata для коротких заголовков и результатов, Golos Text для UI/body,
   Onest для математических пояснений.
4. **Palette:** parchment `#F7F0E5`, carbon `#202C28`, amber `#E77B25`, eucalyptus `#D9E5DF`,
   graphite `#6A746F`; тёплая глубина без sepia-theme.
5. **Signature:** `Поля проверки` — факты диагностики и AI-вердикта появляются на внешнем поле,
   тогда как математика и действие всегда остаются на чистом центральном листе.
6. **Assets:** code-native cropped geometric acorn stamp; одна небольшая белка на completion
   endpaper, ноль персонажей в диагностике и решении.
7. **Layout/rhythm:** page turns только причинные; 32 px reading column rhythm, широкая рабочая
   зона и минимум обрамляющих карточек.

## C — Фокус-станция

1. **Register:** уверенный; запрещены детский game HUD, SaaS onboarding и постоянный coach chat.
2. **Direction/spatial:** каждый этап — одна полноэкранная станция; route живёт в компактном
   chapter switcher, текущая математика занимает центр, действие закреплено в reach-zone.
3. **Typography:** Alumni Sans для крупных коротких stage labels, Onest для текста и математики;
   цифры крупные, проза спокойная.
4. **Palette:** near-white `#FFF9EF`, ink `#172D27`, live orange `#FF8A2B`, apricot `#FFD9B2`,
   teal `#347465`; контрастный event без orange wall.
5. **Signature:** `Линза уверенности` — code-native апертура показывает, что именно доказано
   ответом/фото, и раскрывает следующий этап только после серверного evidence.
6. **Assets:** одна абстрактная squirrel-tail линия в onboarding и success; никаких raster hero,
   mascot отсутствует в task/photo/guided.
7. **Layout/rhythm:** mobile строится вокруг thumb reach и одного stage; desktop использует
   свободное поле + off-center task plane, а не centered phone-card.

## Concept selection rule

Все три направления сначала должны быть renderable на 375×844 и 1280×900 минимум в состояниях
`route`, `photo-task`, `guided`. Выбор делает frozen rubric и два blind reviewer. Палитра не
может компенсировать непонятный следующий шаг или слабую связь с реальными данными.
