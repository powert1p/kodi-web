# Production brief — адаптивная NIS-подготовка с решением по фото

## Задача

Новый ученик должен зарегистрироваться, понять формат математической части отбора NIS,
пройти короткую адаптивную диагностику, увидеть объяснимый персональный маршрут и только затем
явно начать первую подходящую тему. В уроке основной сценарий — решить содержательную задачу
целиком на бумаге и отправить одно фото решения. Пошаговый typed-разбор включается только по
явному запросу ребёнка и завершается новой самостоятельной transfer-задачей по фото.

## Аудитория и job-to-be-done

- Ученик 10–13 лет, впервые открывший приложение для подготовки к поступлению в NIS.
- На каждом экране он должен без исследования интерфейса понимать: где находится, зачем это
  нужно для NIS и какое одно действие сделать сейчас.
- Родитель должен перед первым фото получить отдельный consent handoff; обычная регистрация не
  выдаёт согласие за родителя.

## Неподвижная продуктовая последовательность

1. Регистрация или повторный вход.
2. Короткая адаптация профиля: класс, язык, цель и знакомство с картой подготовки.
3. Честное объяснение формата двух математических блоков NIS в текущем scope продукта.
4. Адаптивная диагностика: один вопрос и одно действие на экран, сохранение после каждого ответа.
5. Результат с доказательствами, без ярлыка «умный/слабый».
6. Персональный маршрут из реальных узлов графа знаний с объяснением первого приоритета.
7. Явное действие «Начать тему».
8. Самостоятельная задача: бумага → одно фото всего решения → мягкий AI-вердикт.
9. Только по явному «Не знаю, как решать» — guided typed/options по той же задаче, без фото шагов.
10. После guided — новая transfer-задача, снова целиком и одним фото.
11. Mastery обновляется только от уверенно проверенной самостоятельной работы; guided-прогресс
    сохраняется отдельно и не выдаётся за освоение.

## Product truth и ограничения

- Текущий пользовательский продукт — React PWA `webapp/` на `/app/`; Flutter — legacy.
- Backend — FastAPI + SQLAlchemy async + PostgreSQL; весь resumable state хранится серверно.
- JWT в browser storage — единственное допустимое клиентское состояние идентичности.
- `0.85` — mastery threshold практики; `0.7` — отдельные diagnostic/exam/UX semantics.
- Фото проверяется grounded относительно verified task/decomposition; код проверяет математику,
  LLM распознаёт структуру и формулирует безопасный feedback.
- Неуверенный или нечитаемый результат никогда не превращается в ложное «неверно».
- NIS scope приложения честно ограничен блоками «Математика» и «Количественные характеристики»;
  приложение не изображает подготовку к языкам и естествознанию.

## Register lock

Регистр: **уверенный**.

Он запрещает:

- dashboard/card soup и каталог функций вместо следующего действия;
- кислотную gamification, reward-first экран и mascot-first hero;
- взрослую корпоративную официальность, экзаменационный страх и «оценку личности»;
- оранжевые стены, glass/AI-gradient и декоративную математику.

Он помогает ребёнку видеть сначала смысл этапа, затем математику, затем одно действие и только
после него спокойное доказательство прогресса.

## Anti-reference

- прямой переход регистрации в `mixtures-1`;
- прежняя модель «Сегодня / закрываем ошибки»;
- worked-example-first и typed-answer-first как default урока;
- фото каждого шага;
- постоянный AI-chat, меню возможностей и mascot рядом с условием;
- старые DOM/composition/screenshots и текущая DS как обязательный визуальный канон;
- точное копирование старого AiPlus lockup или персонажа из костюма.

## Устройства и обязательные состояния

- 375×844 — primary; 1280×900 — самостоятельная desktop-композиция.
- Loading, local error + recovery, empty, success, unreadable photo, uncertain verdict,
  offline/retry with preserved input, resume after reload/relogin, reduced motion.
- Touch target не меньше 44 px, учебный текст не меньше 18 px, видимый keyboard focus,
  `aria-live` для асинхронного результата, отсутствие page-level horizontal overflow.

## Definition of done

Production-ready означает не только новый UI. Обязательны durable onboarding/diagnostic/route/
photo/guided/transfer state, разные маршруты для разных диагностических профилей, реальные
Gemini-вызовы без успешных mock-fallback, автоматические тесты, browser journey matrix,
независимый review и проверенный публичный URL.
