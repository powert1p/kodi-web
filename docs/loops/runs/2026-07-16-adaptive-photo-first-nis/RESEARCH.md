# Research — NIS format и UX-факты

Проверено: **2026-07-16**. Источники формата — только официальные страницы и материалы NIS.

## Официальный формат, который влияет на продукт

- Страница образцов конкурсного отбора:
  <https://www.nis.edu.kz/ru/page/konkurstyq-irikteu-tapsyrmalarynyn-ulgileri>
- Официальная памятка по структуре тестирования:
  <https://www.nis.edu.kz/storage/files/01K9E6B9J358W8M3C06ZC40A62.pdf>
- Официальный образец первого дня:
  <https://www.nis.edu.kz/storage/files/01JSJ1FMQBQMCEGGVJ7ZTC4ESA.pdf>
- Действующие правила конкурсного отбора:
  <https://www.nis.edu.kz/ru/page/konkurstyq-irikteu-erezeleri>

В официальной памятке первый день состоит из математики — 40 заданий / 60 минут,
количественных характеристик — 60 заданий / 30 минут и естествознания — 20 заданий / 30 минут.
Второй день содержит русский, казахский и английский языки. Поэтому текущий продукт не должен
называть свой математический маршрут полной подготовкой ко всему тестированию.

В официальном образце первого дня математика даёт до 400 баллов, количественные характеристики
— до 300, естествознание — до 200. Указаны отдельные минимальные пороги: 140/400 для математики
и 120/300 для количественных характеристик. Эти пороги описывают конкурсный формат и не должны
смешиваться с внутренними BKT-порогами приложения.

Образец проверяет не только вычисления: функциональную грамотность, числовое и логическое
мышление, понимание рисунков, графиков и диаграмм. Значит карта подготовки не может состоять из
ленты одношаговых примеров вроде `63 + 28`.

## Что уже можно использовать в kodi-web

- `nis_knowledge_graph_v01.json`: 114 узлов с prerequisites, сложностью и связями с типами
  экзаменационных заданий.
- `problems_v10.json`: 2525 задач; для photo-first выбирать только задачи с проверенной
  декомпозицией и содержательным многошаговым решением.
- BKT и `Attempt`: база для адаптации, если diagnostic и learning sources разделены корректно.
- `Student.paused_diagnostic`: уже доказывает возможность durable resume, но нужен единый
  journey contract и явный versioned result/route.
- `photo_consent`, grounded trainer diagnosis, image validation и recurring error storage:
  переиспользовать инфраструктуру, не старый UX пофрагментной сдачи.

## UX-принципы из ui-ux-pro-max, вошедшие в frozen gate

- одна primary action на экран и локальная ошибка возле вызвавшего её действия;
- пользовательский ввод сохраняется при network retry;
- асинхронные вердикты объявляются через `role=alert`/`aria-live`;
- цвет не является единственным носителем статуса;
- accessible name у icon-only actions, логичный keyboard order и заметный focus;
- reduced motion сохраняет причинность переходов, а не только отключает animation;
- loading сохраняет геометрию и не подменяется пустым экраном.

Generic-рекомендация поискового скрипта (`liquid glass`, hero/testimonials/CTA, SaaS blue) была
отклонена как не соответствующая учебному приложению и owner feedback.
