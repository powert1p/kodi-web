# Shape concepts — webapp v11

Общий register для всех направлений: **увлечённый**. Контент и размеры одинаковы; различается пространственная система, art direction и signature interaction.

## A — Лента решения

1. **Composition:** mobile строится вокруг одной вертикальной ленты шагов, которая проходит через math-stage; desktop — асимметричная лента 7/5 с задачей слева и живым маршрутом справа.
2. **Hierarchy:** компактный контекст → крупная формула → один текущий шаг → answer/action; никакого hero headline над задачей.
3. **Typography:** Tektur только для коротких action/display меток, Onest для текста и математики.
4. **Color/material:** молочная бумага, чернильный navy, приглушённый шалфей, restrained AiPlus orange; тонкие «сгибы» ленты дают глубину без card soup.
5. **Brand/mascot:** новый `ai+` seed-mark в navigation; белка может физически опираться на конец ленты на Hub и исчезает в active Drill.
6. **Signature:** proof-tape с пронумерованными вырубками; правильный шаг не «зеленеет целиком», а защёлкивает следующий сегмент.
7. **Responsive:** mobile — одна непрерывная траектория; desktop — та же лента растягивается, а не превращается в набор dashboard cards.

## B — Матемастерская

1. **Composition:** экран как аккуратный workbench: центральная рабочая поверхность задачи, узкая rail-панель прогресса и нижняя инструментальная зона ответа; desktop раскрывает рабочее место по горизонтали.
2. **Hierarchy:** текущая операция занимает визуальный центр; navigation и mascot находятся на периферии как инструменты, не как контент.
3. **Typography:** высокий Alumni Sans для коротких разделов/чисел, Onest для body и формул; плотная, но не официальная типографическая шкала.
4. **Color/material:** тёплый white, graphite, dusty blue и copper-orange; небольшие solid-поверхности вместо тёмного полноэкранного stage.
5. **Brand/mascot:** знак собирается из `a`, точки и плюса как tool-stamp; старшая белка появляется на Hub как наставник за краем стола.
6. **Signature:** измерительная rail-линейка: active notch показывает, где именно ученик находится, а success передвигает оранжевый clamp.
7. **Responsive:** на mobile rail становится боковой шкалой и не отнимает высоту; на desktop workspace остаётся единым, без bento-grid.

## C — Орбита навыка

1. **Composition:** формула находится в спокойном центре, а contextual progress проходит по открытой дуге вокруг неё; Hub использует крупную дугу маршрута, Drill — компактное кольцо, Srez — дугу из 12 точек.
2. **Hierarchy:** math центрируется не типографическим гигантизмом, а свободным полем и направлением орбиты; answer закреплён близко к активному узлу.
3. **Typography:** Golos/Onest с мягкими крупными цифрами и очень короткими labels; без display serif и без рекламного headline.
4. **Color/material:** bone, soft lilac-grey, deep plum-ink, pale apricot и один pure orange event; фон светлый и воздушный.
5. **Brand/mascot:** seed-mark становится маленьким orbital spark; белка находится только у стартовой/финальной точки, а не поверх математики.
6. **Signature:** живая orbit trace — отправленный ответ проходит по дуге к следующему узлу; reduced-motion показывает ту же связь статическим change of stroke.
7. **Responsive:** mobile использует обрезанную дугу за пределами viewport, desktop — широкую открытую траекторию, сохраняя один центр внимания.
