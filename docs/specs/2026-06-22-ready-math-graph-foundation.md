## Как адоптировать (гибрид — не выкидывать свою работу)
Готовый граф КРУПНЫЙ (1 стандарт ≈ несколько твоих микро-навыков). Поэтому:
1. **Скачать** `standards.jsonl`, отфильтровать домены 3-6 классов: NF, NBT, OA, MD, G, RP, NS, EE (выкинуть статистику/данные и старшие классы). ~90-130 узлов, несколько сотен стрелок.
2. **Импортировать** стандарты как «макро-узлы» + их стрелки `progress-to/progress-from` как готовый граф пререквизитов.
3. **Подвесить** каждый из твоих 118 узлов / микро-навыков под ОДИН стандарт — они становятся детальным слоем и **наследуют стрелки** сверху. Своих тонких стрелок внутри стандарта — 1-3, дорисовать руками = часы.
4. **Локализовать под НИШ:** игнорировать американские номера классов (брать только ПОРЯДОК), перемапить на классы НИШ; добавить узлы где НИШ тяжелее (устный счёт, текстовые/логика, теория чисел) — из своих 2525 задач.
5. **Сверять спорные стрелки** с бесплатными Common Core Progressions (PDF, обоснование).

Атрибуция: «Prerequisite structure derived from the Common Core Coherence Map (Student Achievement Partners), ODC-By 1.0».

## Честные оговорки (red-team)
- **Гранулярность:** ни один открытый граф не даёт атомарный уровень бесплатно — крупный слой решён адоптом, тонкий ты уже построил (118 узлов + размётка дробей). Это идеальный стык, а не проблема.
- **Фит под НИШ:** математика 3-6 на ~90% культуро-независима (стрелки переносятся), НО номера классов и акценты американские — топологию брать, метки классов и вес тем переделать под реальные прошлые работы НИШ. Эту валидацию не сделает ни один внешний граф — только ты по прошлым вариантам.
- **Усилие:** 1-2 дня дата-работы для соло-владельца, не инженерный проект.
