# Движок лесенок «норм» — DoD-7 без нарушений

**Дата:** 2026-06-24 · **Тип:** доводка движка + переавторинг

## Цель
Свести воедино выводы сессии и довести движок + 7 лесенок до состояния «**DoD-7 владельца выполнен без нарушений**», включая те, что критик раньше проворонил (дробление, двойной вопрос, формула/ответ в варианте). Опереться на находку «Sonnet строже/плотнее Opus» и на research-методологии (author-critic self-refine, human-in-the-loop QC).

## Scope
- Дефолт модели агентов движка → **Sonnet** (`MODEL = _a.model || 'sonnet'`, reversible через args).
- **Усилить критика**: к пунктам DoD добавить явные примеры частых нарушений из самоаудита (п.1 дробление, п.3 двойной вопрос «и», п.4 формула/ответ в опции) + инструкцию «будь придирчив, лучше лишний revise». Зеркально — явные запреты автору.
- Перегон 7 → самоаудит против DoD-7 руками → дочинить остаток → вписать в `srez.ts`.

## Out-of-scope (отдельные задачи)
- Overgenerate-and-rank на дистракторы (методология §5 research) — следующий слой.
- Переписывание DoD на язык ECD (evidence-model) — отдельно.
- Multi-judge голосование — только если sharpened-критик не справится по самоаудиту.
- Grounding из `full_decomposition_v1.json` для всех (сейчас мой grounding; для 4 из 7 совпадает с банком).
- SymPy-гейт на числа.

## Продуктовые решения
- Лесенки = логичные шаги решения по DoD-7; бесшовные 3 задачи (фотик/турист/8:5) остаются.
- Варианты выбора — голые значения/короткие фразы, БЕЗ формулы решения внутри (WP03: `[8 / 4 / 20]`, не `(24−16)÷2`).

## Технические решения (мои)
- Sonnet default: на прогоне 7 автор плотнее (WP05 3 vs 5), критик строже (поймал 3 утечки, что Opus принял), быстрее/дешевле (77с/369k vs 120с/415k). Reversible.
- Критик sharpened single (дешевле multi-judge); эскалация на 2-й проход только если самоаудит найдёт пропуск.
- Human-in-the-loop самоаудит — узаконенная часть методологии QC (AIG не заменяет эксперта), не костыль.

## Критерии успеха
- Мой самоаудит 7 лесенок против DoD-7 = **0 нарушений** (каждый пункт по каждой задаче).
- `tsc + vite build` зелёный; Playwright ≥2 сквозняком до closure + проверка: нет двойных вопросов, варианты голые, нет утечек вычисления в формулировку; консоль 0 ошибок.

## Риски
- Sonnet-критик всё ещё пропустит нарушение → ловлю самоаудитом руками (методология это предусматривает).
- Дефолт-Sonnet может быть хуже на иных задачах → reversible через `args.model`.
