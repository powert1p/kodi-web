# Движок лесенок v2 — заземление на методологию (ECD/AIG) + verified-банк

**Дата:** 2026-06-24 · **Статус:** в работе

## Цель
Прошлый движок улучшал лесенки одним приёмом из research (усиленный критик), а **7-пунктовый DoD я ошибочно взял за фундамент** — это был лишь пример «здравого смысла» от владельца, не методология. Перестроить движок так, чтобы ФУНДАМЕНТОМ были методики из deep-research, и они были применены НА ДАННЫХ, которые уже есть в банке `docs/specs/full_decomposition_v1.json` (2525 задач, 94% с реальными ошибками учеников). Доказать на прогоне 50 задач (Opus).

## Что из research применяю (и на каких данных)
- **AIG / когнитивная модель** (Gierl): лесенка = ПРОВЕРЕННАЯ декомпозиция `steps[]` (SymPy-verified), а не импровизация LLM. Структуру шагов не выдумываем.
- **KLI / Q-matrix** (Koedinger): у каждого `steps[].micro_skill` уже проставлен KC. Каждая ступень нацелена ровно на ОДИН micro_skill.
- **ECD evidence model** (Mislevy): ступень = задача, добывающая СВИДЕТЕЛЬСТВО владения её micro_skill. Ответ ученика должен различать «владеет / не владеет». Это и есть переформулировка бывших «логических» правил — не аксиомы, а следствие.
- **Overgenerate-and-rank дистракторы** (Scarlatos/DiVERT): варианты `choose` НЕ выдумываются — берутся из реальных `fingerprints[]` (wrong_answer + mistake_ru). Стадия: из всех fingerprints породить кандидаты-дистракторы для нужной ступени → ранжировать по диагностичности → top-1..2 + верный.
- **Author–critic self-refine + validation** (MCQG-SRefine/GradeOpt): critic-стадия валидирует против ECD-чеклиста; ≤1 правка.

## Scope
- Переписать `author-prompt.md` + `critic-prompt.md`: фундамент = ECD/AIG/Q-matrix; бывший DoD-7 → «признаки хорошего свидетельства», производные, а не закон.
- Новая стадия `distractor` (overgenerate-and-rank из `fingerprints[]`) в `ladder-engine.workflow.js`. Pipeline: author → distractor → critic → revise(≤1).
- Модель агентов: **Opus** (`args.model` override сохранить).
- Скрипт сборки 50 task-records из verified-банка (spread по 16 доменам, ≥1 fingerprint, 2–5 шагов).
- Прогон 50 на Opus → JSON результатов + отчёт качества в чат.

## Out-of-scope (якорь от расползания)
- НЕ трогаю кабинет/`srez.ts` (7 задач) — это валидация движка, не продуктовая поверхность (решение владельца).
- НЕ ввожу SymPy-гейт на числа лесенки (числа и так из verified `steps[]`).
- НЕ строю n-layer item-models / генерацию НОВЫХ задач — только лесенки к существующим.
- НЕ мигрирую DoD в формальный psychometric-язык дальше, чем нужно агенту.

## Технические решения (мои, 1 строка каждое)
- Дистракторы из данных, а не из LLM-фантазии: fingerprint = реальная ошибка → диагностичнее выдуманной (named weak-spot из прошлого раунда). Fallback: если <1 fingerprint — задачу в выборку не берём.
- Стадия distractor отдельным агентом (не внутри author): чистое разделение overgenerate-rank от построения лесенки; легче ревьюить.
- Opus дефолтом на этот прогон (директива владельца); прошлый sonnet-вывод был под СТАРЫМИ промптами — перепроверим под новыми, отметим в отчёте.
- Результаты — JSON в `scratchpad` + сводка в чат (не папка/PDF: правило «reports только по запросу»).

## Критерии успеха (проверяемые)
- `node t" --check` (синтакс) движка чистый; прогон 50 завершается без падения pipeline.
- ≥90% choose-ступеней имеют дистрактор, привязанный к реальному `fingerprint` (не выдуманный) — считаю по выдаче.
- Critic-вердикт + мой глаз-аудит на выборке 8/50: 0 ECD-нарушений (один KC/ступень, нет утечки ответа в вопрос/варианты, доводит до ответа).
- Отчёт в чат: % accept, % с fingerprint-дистрактором, топ-режимы провалов, 2–3 примера лесенок (хороший + спорный).

## Риски
- Fingerprint = неверный ФИНАЛ, а choose-ступень промежуточная → дистрактор на этой ступени надо вывести из mistake_ru. Замечу: если вариант = голое неверное значение, согласованное с mistake_ru. Иначе critic ловит «выдуманный дистрактор».
- Opus может оказаться многословнее/слабее sonnet под новыми промптами → отмечу в отчёте, дефолт оставлю настраиваемым.
- 50 задач × 4 стадии на Opus — большой прогон; caps Workflow (1000 агентов/16 конкур.) держат.
