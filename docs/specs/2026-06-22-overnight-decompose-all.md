# ТЗ: Ночной прогон — разметить ВСЕ задачи + полный каталог микро-навыков

> Дата: 2026-06-22 · Автономный ночной прогон под `/goal`. Продолжение пилота дробей (docs/specs/2026-06-22-fractions-pilot-error-trainer.md).

## Цель (что должно получиться к утру)
Прорешать и разметить **все ~2525 задач** так же, как дроби: разбить на шаги, каждый шаг → один атомарный микро-навык, каждое число проверить Python/SymPy, посчитать отпечатки ошибок для многошаговых. Затем **собрать единый каталог микро-навыков по всему курсу и посчитать, сколько их вышло** (главный вопрос владельца — решаем «че делать» по факту числа).

## Проверяемое конечное состояние (для `/goal` — это и есть условие остановки)
Готово ⟺ ВСЕ три файла существуют и проходят пороги:
1. `docs/specs/full_decomposition_v1.json` — декомпозиции **≥ 2500 из 2525** задач; у каждой: steps[{n, instruction_ru, micro_skill, expected_value, verified}], fingerprints, primary_micro_skill, needs_review.
2. `docs/specs/full_micro_skill_catalog.md` — единый каталог микро-навыков с **итоговым числом** + разбивкой по 15 разделам + частотой каждого навыка.
3. В JSON: **≥ 90% числовых шагов = sympy_ok** (поле stats), и список `needs_review` собран.

## Scope
- Разметка всех 2525 задач (дроби 248 уже сделаны в `fractions_steps_v1.json` — переиспользовать/слить).
- Единый каталог: навыки выводятся из разметки (snake_case код + ru-метка), затем консолидация синонимов в канонический список.
- Подсчёт: сколько микро-навыков всего, сколько новых vs переиспользуемых, по разделам.

## Out-of-scope (ночью НЕ делать)
- НЕ трогать тесты (срез/воскресный/интенсив), не интегрировать во флоу.
- НЕ менять прод-код, схему БД, миграции.
- НЕ строить граф/тренажёр — только разметка + каталог + счётчик. Решение «че дальше» — утром с владельцем.
- DROP/DELETE/TRUNCATE данных — запрещено.

## Механизм (как исполнять — идемпотентно, с продолжением)
1. Вход: `/tmp/kodi_pilot/all_problems_input.json` (2525 задач: idx, node_id, tag, topic, text_ru, answer, answer_type, difficulty, solution_ru). Если файла нет — пересоздать из `backend/data/problems_v10.json` + графа.
2. Запустить workflow `kodi-decompose-all` (Workflow tool): параллельные агенты размечают чанки, **проверяя каждое число реальным Python** (`from fractions import Fraction`), grounding на `solution_ru`. Seed-словарь = каталог дробей + арифметика; новые навыки предлагать snake_case.
3. Консолидация: собрать все уникальные предложенные навыки (код+метка+частота), слить синонимы в канонический каталог.
4. Сборка: записать оба файла + независимая перепроверка (финальный шаг == ответ) + stats.
5. **Идемпотентность/продолжение:** если `full_decomposition_v1.json` уже частичный — доразметить только недостающие idx, не переделывать готовое. На каждом ходе `/goal` проверять условие остановки выше.

## Критерии успеха (проверяемые)
- Оба файла на диске, пороги (≥2500 задач, ≥90% sympy_ok, каталог с числом) выполнены.
- Независимая сверка финал-шаг==ответ ≥ 99% (как на дробях было 100%).
- Каталог: одно итоговое число микро-навыков + разбивка по разделам.

## Риски
- SymPy не покроет текстовые/нечисловые шаги (word_problems 273, logic 260) → помечать `non_numeric`/`llm_only`, не падать; такие — в needs_review.
- Часть агентов упадёт на большом объёме → собрать частичный результат, недостающие idx доразметить следующим ходом (продолжение).
- Каталог раздуется (синонимы) → обязательная консолидация; цель ~120-180 навыков, не тысячи.
- Объём большой (~115 агентов) → ок для ночи; стоимость не ограничение (Haiku + батч).
