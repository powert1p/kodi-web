# ТЗ: настоящий MVP — поэтапная подготовка к НИШ

**Дата:** 2026-07-10 · **Определение MVP от владельца:** «ребёнок начал учиться и было полезно». Не демо тренажёра ошибок, а учебный путь.

## Диагноз (фидбек владельца, протык флоу 2026-07-10)
1. **Срез — говно**: задачи слишком простые (выбор difficulty ASC = «23+45» для кандидата в НИШ). Уровень не учитывается вообще.
2. **Нет пути**: после среза только «разбирай ошибки». Учиться — то есть проходить темы по нарастающей — негде.
3. **Бот тупой**: тьютор отвечает поверхностно, не опирается на метод решения.
4. **Два места отправки решения**: ступень с вводом + нижний «Сфотографировать решение» — ребёнок путается.
5. **Нет теории/советов**: ни «как решать», ни метода перед задачей.
6. **Кнопки неудобные** + заглушки (фейковые стрик «5» и «1 280» очков).

## Целевой флоу MVP (ребёнок, 30 минут, полезно)
Регистрация (+ класс) → онбординг-срез ПО УРОВНЮ → маршрут из КТП-блоков («ты в Блоке 3: Делимость — закрыто 2 из 4 узлов») → в узле: карточка «Как решать» (метод+пример) → серия задач с ростом сложности (ввод или «По тетради» фото-шагами) → ошибки уходят в разбор с умным Кёди → блок закрыт → следующий.

## Заходы

### Заход 1 — убрать стыдное (СЕЙЧАС)
- **1a. Класс при онбординге**: селект «Иду в 5/6/7 класс» (students.grade уже есть в схеме, регистрация его не пишет). По классу — целевое окно difficulty.
- **1b. Срез v2**: задачи в окне сложности класса (готовлюсь в 7 → difficulty 3-5, а не 1), разброс тем сохраняем, 2 задачи-стретча сверху. Задачи «23+45» новичку-семикласснику больше не показываем.
- **1c. Один вход сдачи**: нижний PhotoCapture из drill убрать; фото-путь = только режим «По тетради» (пошаговый). Один экран — один способ сдать.
- **1d. Теория «Как решать»**: карточка метода на каждый из 114 узлов (метод 3-6 строк + мини-пример + типичная ловушка; генерация батчами + моя выборочная приёмка) → nodes.theory_ru → кнопка «Как решать?» в drill и перед серией. Тон — Кёди, без канцелярита.
- **1e. Тьютор-апгрейд**: в промпт добавить theory-карточку узла + текущую ступень + жёсткие правила (отвечать по методу, ≤3 предложения, встречный вопрос, никогда не лить воду). Модель та же (flash), сила — в grounding.
- **1f. Кнопки/заглушки**: убрать фейковые стрик/очки (показывать реальные или ничего); Playwright-проход по флоу на 390px — починить очевидные неудобства таргетов/порядка CTA.

### Заход 2 — учебный маршрут (следующий)
- Экран «Мой путь»: 13 блоков КТП, прогресс узлов, текущий блок раскрыт.
- Практика узла: серия 5-8 задач difficulty по нарастающей, mastery-гейт 0.7/0.85, интеграция с «По тетради».
- Срез при завершении блока (повторение) + анти-забывание.

### Заход 3 — пилотная полировка
- Родительский прогресс (ссылка/выгрузка), пуш-напоминания (PWA), ru/kz.

## Out-of-scope (не сейчас)
Планшеты/канвас (Этап 3), голос, файнтюн, teacher-дашборд, новая эстетика (канон v5 не трогаем).

## Критерии приёмки Захода 1 (проверяемые)
- Свежий аккаунт «иду в 7» получает срез: ≥80% задач difficulty ≥3, 0 задач difficulty 1.
- В drill ровно ОДНА точка отправки решения в любом режиме.
- У 114/114 узлов есть theory_ru; в drill открывается «Как решать?»; выборочные 10 карточек приняты владельцем контента (мной).
- Тьютор в e2e-диалоге: отвечает по методу узла, ≤3 предложения, задаёт встречный вопрос, не раскрывает ответ.
- Стрик/очки: либо реальные числа из БД, либо скрыты.
- Гейты: pytest + vitest + lint:design + build + Playwright-скрины 390px (срез/drill/hub) + live-проверка на проде свежим аккаунтом.
