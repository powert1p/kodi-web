# Спека: движок авторинга reasoning-лесенок

> Дата: 2026-06-23 · /brainstorming · Ultracode-ресёрч (KLI/knowledge-components, expertise reversal, expert blind spot, diagnostic questions, anti-template prompting).

## Цель
Перестать делать задачи руками «по шаблону» (один скелет, два вопроса в пункт, leading-вопросы, простыни-варианты). Завести **движок**: на вход — сырая задача из базы + эталонные шаги; на выход — качественная reasoning-лесенка. Автор — всегда агент, не руки. Качество держит промпт + агент-критик + моя выборочная проверка.

## Scope
- `engine/author-prompt.md` — промпт агента-автора (принципы + контрастные ok/не-ok РАЗНОЙ формы, без канонического скелета).
- `engine/critic-prompt.md` — промпт агента-критика (поштучный чеклист, цитата+правка, вердикт accept/revise/needs_human).
- `engine/ladder-engine.workflow.js` — оркестратор: автор → критик → (≤1 правка) на задачу, читает промпты из args.
- Обкатка на 3 НИШ-задачах (фотоаппарат, отношения, свежая авто/автобус) + моя проверка результата.

## Out-of-scope (якорь)
- НЕ прогоняем все 2525 задач (это overnight-батч потом).
- НЕ трогаем backend/схему/Flutter; рантайм-эндпоинт и таблицу `rungs` — отдельным заходом.
- НЕ собираем реальные wrong_answer детей (пока нет данных) — дистрактор по эвристике «частая ошибка навыка», provenance=heuristic.

## Продуктовые решения (бриф ясен, вопросов не было)
- Движок офлайн-батч (как существующая декомпозиция): LLM авторит, критик+человек проверяют, рантайм только читает frozen-лесенки.
- Источник правды по математике — эталонные `steps[]` из full_decomposition_v1.json (SymPy-проверены); LLM делает ТОЛЬКО педагогику, не переизобретает арифметику.

## Технические решения (мои, обоснование в строку)
- **Grounding на steps[]** — каждый шаг = один micro_skill с проверенным числом; автор конвертирует подмножество в Socratic-ступени → математика наследуется, врать негде.
- **Анти-шаблон в промпте, не температурой** — нет названного «скелета»; few-shot разной формы (2/3/4 ступени) с пометкой «образцы разброса»; форма выводится из reasoning (где сломался ребёнок). Температуру/penalty — вторично.
- **Автор↔критик ≤2 проходов**, дальше needs_human; критерий C8 у критика прямо ловит «форма как обобщённый шаблон».
- **Промпты в .md, движок читает из args** — единый источник правды, владелец крутит промпт правкой .md.
- **Скретч-поля** (`reasoning`, `_kc`, `_targets`) для критика, стрипаются при freeze; `_kc` оставить тегом (связь с тренажёром по micro_skill).

## Критерии успеха (проверяемые)
- 3 задачи прогнаны; на каждой лесенка проходит рубрику (критик verdict=accept или после 1 правки).
- Формы РАЗНЫЕ (не один скелет): разное число ступеней и/или разный тип первой ступени.
- Фотоаппарат: ушёл перебор (≤3 ступени, одна choice про базу, вопрос не называет ответ).
- Отношения: введены через «доли», без холодной нотации `2k`.
- Свежая (авто/автобус): осмысленная лесенка без моего ручного дизайна — доказательство, что движок работает на незнакомой задаче.
- Я читаю результат глазами (рубрика) + при желании рендерю в кабинете.

## Риски
- Критик пропустит leading/дистрактор-не-та-ошибка (самые незаметные) → мой спот-чек + банк BAD растёт от reject.
- Без реальных wrong_answer дистрактор может быть не «той» ошибкой → provenance=heuristic, перегенерим по реальным срезам позже.
- Шаблонность — кросс-задачное свойство, поштучный критик не видит → смотрю 3 лесенки рядом на разнобой.
