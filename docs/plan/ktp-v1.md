# КТП v1 — структура обучения «учитель + AI» (мультикласс)

**Дата:** 2026-07-06 · **Основание:** граф v03 (аудит `docs/specs/2026-07-06-graph-topic-audit-verdict.md`), порядок блоков проверен против рёбер программно (0 нарушений). **Формат:** универсальные учебные блоки БЕЗ привязки к неделям — центр растягивает под своё расписание (1 блок ≈ 1-3 занятия теории + AI-практика между ними; Б9 — длинный, 4-6 занятий).

## Как это работает (цикл блока)

1. **Вход в блок — диагностика, не календарь.** Новый ученик проходит полную диагностику (Flutter) или мини-срез (PWA `/srez`); AI знает mastery по каждому узлу. Ученик стартует с первого блока, где есть незакрытые узлы — мультикласс означает, что группа может идти по теории вместе, а AI-практика у каждого своя.
2. **Теория — учитель.** Занятие в центре по акцентам блока (ниже). Учитель видит из выгрузки (аналитика/CSV), какие узлы блока у группы проседают.
3. **Практика — AI, между занятиями (дома или в центре):**
   - **Разбор ошибок** (hub → drill): лесенка шагов, подсказки Кёди, чат-тьютор. Источник ошибок — срез и практика.
   - **«По тетради» (поэтапная сдача)** — основной режим домашней работы: ребёнок решает в тетради, фотографирует КАЖДЫЙ шаг, AI сверяет (match / mismatch / «сфотай ещё раз» — unsure не наказывается). Это одновременно сбор датасета.
   - **Закрытие** (verification): контрольная задача по узлу → ошибка помечается закрытой, прогресс темы растёт.
4. **Гейт выхода из блока:** все узлы блока «закрыты» в графе (mastery ≥ 0.7 UX-порог; для экзамен-трека — 0.85 BKT в практике) И нет открытых ошибок по узлам блока в тренажёре. Не прошёл — AI даёт climb-down (более простые декомпозиции), учитель получает сигнал.
5. **Повторение:** каждые 2-3 блока — мини-срез по пройденному (12 задач) для выявления забывания; финал программы — прогон по 15 экзаменационным вершинам (exam-режим диагностики).

## Роли и границы

- Учитель объясняет НОВОЕ и разбирает типовые ловушки; AI не вводит теорию — он тренирует, диагностирует и закрывает ошибки.
- Ошибку ученику сообщает Кёди (персонаж), не «система»; false-reject хуже пропуска — сомнительное фото = «покажи ещё раз», не «неверно».
- Родителю — согласие на фото ДО первой фотографии (уже в продукте), выгрузка прогресса — владельцу (CSV).
